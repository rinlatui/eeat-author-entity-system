# Experience Signal Layer

## Purpose

Experience signals show that content is informed by practical work, observations, testing, or direct involvement.

## Key Questions

- Is the information visible to users?
- Is the identity consistent?
- Is the source or profile verifiable?
- Does schema support visible content?
- Does the page help users understand responsibility and credibility?
- Can this be audited and maintained over time?

## Recommended Output

```text
Current status:
Gaps:
Priority fixes:
Owner:
Re-test date:
```

## Practical Notes

- Avoid fake trust signals.
- Keep names and URLs consistent.
- Use author and reviewer information to help users.
- Validate schema after implementation.
- Document changes in an audit log.
